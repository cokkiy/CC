# CC项目开发计划

## 项目概述
CC项目是一个基于Rust生态构建的IoT设备和工作站集中管理控制系统，旨在实现从100台设备到10W+ IoT设备的可扩展管理。

## 当前状态分析

### 已完成工作
1. **基础架构迁移**：从Qt/C++迁移到Rust/Tauri架构
2. **核心组件**：
   - CC-rStationService：工作站服务（Rust）
   - CC-rClient：客户端（Tauri + React）
   - 已完成网络遥测、进程监控等基础功能
3. **Phase 3-5任务**：正在进行调试日志添加和数据流测试

### 架构问题识别（根据架构审查报告）
1. **传输层问题**：gRPC不适合1:N订阅场景，需要替换为MQTT
2. **数据格式问题**：Protobuf不支持动态扩展，需要JSON/CBOR
3. **客户端扩展性问题**：单体架构，无插件系统
4. **中间件缺失**：无消息代理，客户端直接连接工作站

## 技术选型建议

### 消息中间件：**NATS**
- 理由：单一二进制，零依赖，性能优秀，Rust生态支持好
- 替代方案：EMQX（功能更丰富但更重）

### 数据格式：**CBOR + JSON**
- 理由：CBOR体积小适合IoT，JSON易调试，两者可自动协商

### 插件系统：**libloading + WASM**
- 理由：libloading支持动态加载，WASM提供安全沙箱

### 数据库：**SQLite + sqlx**
- 理由：嵌入式，零配置，Rust生态支持好

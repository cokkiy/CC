# CC Project UI Layout Design

## 1. 概述

本文档为 CC 项目设计一个更美观、简洁、易用的界面布局方案。设计方案参考了以下主流 IoT 平台和监控工具的界面设计：

- **ThingsBoard**: 仪表盘 widgets 网格布局，支持响应式设计
- **Home Assistant**: 卡片式仪表盘，Section 视图，拖拽排列
- **Grafana**: 面板式布局，行/面板结构，支持层级钻取
- **Node-RED Dashboard 2.0**: 页面-组-控件层级结构，12 列网格布局

## 2. 设计原则

### 2.1 视觉层次
- **信息层次清晰**: 重要信息优先展示，次要信息可展开查看
- **状态一眼可辨**: 通过颜色、图标、位置快速识别设备状态
- **减少认知负担**: 遵循 "Less is More" 原则，界面简洁不杂乱

### 2.2 交互设计
- **一致性**: 相同操作有相同的交互反馈
- **即时反馈**: 用户操作后有明确的视觉反馈
- **容错性**: 危险操作需要确认，防止误操作

### 2.3 响应式设计
- **桌面优先**: 适配 1920px+ 为主，1366px+ 可用
- **平板适配**: 1024px-1919px 自适应布局
- **移动端考虑**: 提供最小化视图用于紧急查看

## 3. 整体页面结构

### 3.1 页面分类

CC 项目分为以下主要功能区域（页面）：

```
┌─────────────────────────────────────────────────────────┐
│  顶部栏 (Top Bar)                                         │
│  [Logo] [导航标签] [搜索] [通知] [用户菜单]                  │
├─────────────────────────────────────────────────────────┤
│                                                          │
│  主内容区 (Main Content)                                  │
│                                                          │
│  根据当前页面显示不同内容                                   │
│                                                          │
│                                                          │
├─────────────────────────────────────────────────────────┤
│  状态栏 (Status Bar) - 可选                               │
│  [连接状态] [最后更新] [版本信息]                            │
└─────────────────────────────────────────────────────────┘
```

### 3.2 导航结构

| 页面 | 路径 | 功能描述 |
|------|------|----------|
| **仪表盘** | `/dashboard` | 系统概览，核心指标，告警概览 |
| **设备列表** | `/stations` | 工作站列表，状态监控，批量操作 |
| **设备详情** | `/stations/:id` | 单个设备详细信息，实时数据 |
| **分组管理** | `/groups` | 设备分组管理，批量控制 |
| **脚本管理** | `/scripts` | 脚本列表，创建，编辑，执行 |
| **执行历史** | `/history` | 脚本执行历史，结果查看 |
| **消息日志** | `/logs` | 系统日志，操作日志 |
| **告警中心** | `/alerts` | 告警列表，处理历史 |
| **设置** | `/settings` | 系统设置，用户偏好 |

## 4. 各页面布局设计

### 4.1 仪表盘页面 (Dashboard)

**目的**: 提供系统全局视图，快速了解整体状态

**布局设计**:
```

│  [时间范围选择] [刷新按钮] [布局切换] [全屏]                  │
├─────────────────────────────────────────────────────────┤
│  概览卡片区 (Overview Cards)                               │
│  ┌──────────┐ ┌──────────┐ ┌──────────┐ ┌──────────┐   │
│  │ 在线设备  │ │ 离线设备  │ │ 告警数量  │ │ CPU 峰值  │   │
│  │   12     │ │    2     │ │    3     │ │   78%    │   │
│  └──────────┘ └──────────┘ └──────────┘ └──────────┘   │
├─────────────────────────────────────────────────────────┤
│  主监控区 (Main Charts)                                   │
│  ┌─────────────────────────┐ ┌─────────────────────────┐│
│  │      CPU 使用率趋势      │ │     内存使用率趋势       ││
│  │                         │ │                         ││
│  │                         │ │                         ││
│  └─────────────────────────┘ └─────────────────────────┘│
│  ┌─────────────────────────┐ ┌─────────────────────────┐│
│  │      网络流量趋势        │ │     磁盘使用率趋势       ││
│  │                         │ │                         ││
│  │                         │ │                         ││
│  └─────────────────────────┘ └─────────────────────────┘│
├─────────────────────────────────────────────────────────┤
│  设备状态列表 (Station Status List)                      │
│  ┌─────────────────────────────────────────────────────┐│
│  │ [设备名称] [状态] [IP] [CPU] [内存] [最后活跃] [操作]   ││
```

**设计要点**:
- 概览卡片使用图标+数字+标签形式，状态色区分
- 图表采用时间序列折线图，支持缩放
- 设备状态列表每行是一个设备，点击可跳转详情

## 5. 组件设计规范

### 5.1 颜色系统

```css
/* 状态颜色 */
--status-online: #1f9d55;      /* 在线/成功 */
--status-warning: #c78a00;      /* 警告 */
--status-error: #d64545;        /* 错误/离线 */
--status-info: #2d8cf0;         /* 信息 */

/* 功能颜色 */
--primary-color: #2d8cf0;       /* 主色调 */
--secondary-color: #9b59b6;      /* 次要色 */
--success-color: #1f9d55;        /* 成功 */
--danger-color: #d64545;         /* 危险 */

/* 背景颜色 */
--bg-primary: #0f1419;           /* 主背景 */
--bg-secondary: #1a1f26;         /* 次要背景 */
--bg-card: #242a33;              /* 卡片背景 */
--bg-hover: #2d3640;             /* 悬停状态 */

/* 文字颜色 */
--text-primary: #ffffff;         /* 主要文字 */
--text-secondary: #a0a8b3;       /* 次要文字 */
--text-muted: #6b7280;           /* 弱化文字 */

/* 边框颜色 */
--border-color: #2d3640;
--border-hover: #3d4654;
```

### 5.2 状态指示器

| 状态 | 颜色 | 图标 | 描述 |
|------|------|------|------|
| 在线 | `#1f9d55` (绿色) | ● | 设备正常运行 |
| 警告 | `#c78a00` (黄色) | ⚠ | 需要注意 |
| 离线 | `#d64545` (红色) | ○ | 连接断开 |
| 未知 | `#6b7280` (灰色) | ? | 状态未知 |

### 5.3 通用组件

#### 卡片组件 (Card)
- 圆角: 8px
- 内边距: 16px
- 阴影: 0 2px 8px rgba(0,0,0,0.3)
- 悬停: 轻微上浮 + 边框高亮

#### 按钮组件 (Button)
- 主按钮: 实心背景，主色调
- 次按钮: 边框样式
- 危险按钮: 红色背景
- 尺寸: 小(28px) / 中(36px) / 大(44px)
- 圆角: 6px

#### 输入框 (Input)
- 高度: 36px
- 背景: `--bg-secondary`
- 边框: 1px solid `--border-color`
- 聚焦: 边框变为主色调

#### 表格 (Table)
- 表头: 背景 `--bg-secondary`，文字加粗
- 行高: 48px
- 斑马纹: 奇数行略微深色
- 悬停: 行背景高亮

#### 标签页 (Tabs)
- 下划线样式
- 选中: 主色调下划线
- 悬停: 文字变亮

## 7. 响应式断点

| 断点 | 宽度范围 | 布局调整 |
|------|----------|----------|
| XL | 1920px+ | 完整布局，最大化显示 |
| LG | 1280px-1919px | 完整布局，略微收紧 |
| MD | 960px-1279px | 侧边栏可折叠，网格调整为2列 |
| SM | 600px-959px | 侧边栏收起为图标，单列布局 |
| XS | <600px | 简化导航，表格改为卡片列表 |

## 8. 动画和过渡

- **页面切换**: 淡入淡出，200ms ease-out
- **面板展开/收起**: 高度过渡，300ms ease-in-out
- **按钮悬停**: 背景色过渡，150ms
- **通知出现**: 从右侧滑入，300ms
- **加载状态**: 脉冲动画或旋转图标

## 9. 页面功能清单

## 10. 实施建议

### 10.1 渐进式改造

由于 CC 项目已有一定规模的代码，建议渐进式实施新布局：

1. **第一阶段**: 创建新的页面路由结构和布局框架
2. **第二阶段**: 实现仪表盘页面，作为新布局的展示
3. **第三阶段**: 迁移设备列表到新布局
4. **第四阶段**: 完善其他页面

### 10.2 组件复用

- 提取通用组件：`Card`, `Button`, `Input`, `Table`, `Tabs`, `Modal` 等
- 建立组件库文档
- 保持现有 `DynamicPanel` 机制用于可配置面板

### 10.3 状态管理

- 全局状态: 用户信息、系统配置、主题设置
- 页面状态: 路由参数、筛选条件、分页状态
- 数据状态: 设备列表、告警列表、脚本列表
- 推荐使用 React Context + useReducer 或 Zustand

### 10.4 性能考虑

- 列表虚拟滚动 (react-window/virtualized)
- 图表数据采样
- 懒加载非首屏组件
- 缓存策略优化
